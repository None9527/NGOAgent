# 文件说明

本项目为 NGOAgent 的核心服务，在本次更新中我们在 `internal/domain/service/run.go` 中新增了 `resizeImageIfLarge` 函数，用于修复多模态大图片输入导致 Token 超限报错的问题。

## 详细修改内容
- 引入了 `golang.org/x/image/draw` 并修改了 `run.go` 中的图片解析逻辑。
- 在构建传给大语言模型的消息载荷时（`buildUserMessage`），如果输入图片过大（从分辨率判断，或大于 2MB），系统将强制对其进行双线性等比例缩放（限制最长边为 1024 像素），并采用 JPEG（Quality: 85）格式重新压缩编码。
- 只有 `.svg` 和 `.gif` 类型的图片会被豁免以保留原始矢量或动画效果，其余所有超大的 WebP, PNG, JPEG 都将受到缩放约束。
- 这一机制确保了 `base64` 编码的图片字符串长度被安全地控制在模型的合法范围内（彻底根除了超过 2000 万字节被 API 拒绝引发的报错）。
