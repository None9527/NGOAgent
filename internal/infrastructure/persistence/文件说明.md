# persistence 包 — 文件说明

SQLite + GORM 持久化层。核心数据通过 `conversations.ID` 关联，运行态和分析态按各自合同管理。

## 表结构

```
conversations
  ├── messages → message_tool_calls / message_attachments
  ├── artifacts
  ├── agent_runs → run_checkpoints / run_waits / run_events
  ├── worker_transcripts
  ├── session_token_usages
  └── evo_traces → evo_evaluations → evo_repairs
```

## 文件

| 文件 | 职责 |
|------|------|
| `db.go` | SQLite 连接 + `Conversation` 模型 + WAL 模式 |
| `repository.go` | Conversation CRUD + runtime / analytics 清理 |
| `history.go` | 规范化消息表 `messages` 的读写适配层 |
| `run_snapshot.go` | runtime 表 `agent_runs / run_checkpoints / run_waits` 的主持久化 |
| `migrations.go` | 统一 migration、旧表回填、删除旧表 |
| `transcript.go` | `WorkerTranscript` — Subagent 工具调用追踪 |
| `token_usage.go` | `SessionTokenUsage` — 每 session 的 token 成本 |
| `evo.go` | `EvoTrace/Evaluation/Repair` — 进化自评估链 |
